# SkillForge - Сучасна Платформа для Онлайн Навчання (Бакалаврська робота)

**SkillForge** — це сучасна, високопродуктивна платформа для онлайн-освіти, розроблена з використанням мікросервісної архітектури на базі Spring Boot 3 та React 19. Цей проєкт створений у рамках бакалаврської дипломної роботи.

Платформа забезпечує повний цикл управління курсами, модулями, уроками та квізами (тестуванням). Вона підтримує різні ролі користувачів: **Learner** (Студент), **Educator** (Викладач) та **Curator** (Куратор/Адміністратор).

## 🏗 Архітектура (Microservices)

Проєкт розділений на незалежні мікросервіси для забезпечення масштабованості та ізоляції:

1. **API Gateway (Порт 8080)**: Єдина точка входу для фронтенду. Обробляє CORS, маршрутизує запити до відповідних мікросервісів та ховає внутрішню топологію мережі.
2. **User Service (Порт 8082)**: Відповідає за аутентифікацію (JWT), авторизацію та управління користувачами (ролі, профілі). Містить `DataInitializer` для генерації тестових користувачів.
3. **Course Service (Порт 8083)**: Основний бізнес-домен. Керує життєвим циклом курсів, модулів, уроків, квізів та обробляє логіку Enrollment (записів на курси).
4. **Core Service (Порт 8081)**: Сервіс для спільної логіки або інфраструктурних завдань (заділ на майбутнє розширення).

## 💻 Технологічний стек

- **Backend**:
  - Java 17
  - Spring Boot 3.4.5
  - Spring Cloud Gateway (WebFlux)
  - Spring Security (JWT)
  - Spring Data JPA + Hibernate
  - H2 Database (In-memory, для локального розгортання та тестування)
  - Maven

- **Frontend**:
  - React 19 + TypeScript
  - Vite (Build Tool)
  - Material UI (MUI v6) + Emotion (Стилізація)
  - Framer Motion (Анімації)
  - Axios (Клієнт HTTP)
  - React Router DOM (Навігація)

## 🌟 Основний функціонал

- **Мульти-організаційність (Multi-tenancy)**: Підтримка ізольованих організацій зі своїми кураторами, викладачами та курсами.
- **Ієрархія ролей (RBAC)**: Глобальний `SUPER_ADMIN` для управління всіма організаціями, локальні `ORG_ADMIN` (Адміністратор організації) та `STANDARD` куратори.
- **JWT Аутентифікація**: Безпечний міжсервісний доступ із передачею контексту організацій всередині токену.
- **Управління курсами**: Викладачі та куратори можуть створювати курси в межах своїх організацій, наповнювати їх модулями та уроками.
- **Модуль тестування (Quizzes)**: Вбудовані квізи до уроків для перевірки знань з автоматичним підрахунком результатів.
- **Enrollment**: Студенти можуть записуватися на курси ("Start Course"), а викладачі можуть вручну додавати студентів на свої курси.
- **Сучасний UI/UX**: Адаптивний, анімований та інтуїтивно зрозумілий інтерфейс у стилі Glassmorphism.

## 🚀 Як запустити локально

Для зручності було створено автоматизований скрипт, який запускає всі 4 мікросервіси та React-фронтенд в окремих вікнах.

### Крок 1. Запуск
У корені проєкту запустіть файл:
```bat
.\run_script.bat
```
*Скрипт автоматично запуститься, почекає 10 секунд для ініціалізації мікросервісів і підніме API Gateway та Frontend.*

### Крок 2. Відкриття в браузері
Фронтенд буде доступний за адресою: [http://localhost:5173](http://localhost:5173)

### Крок 3. Тестові користувачі
При старті `user-service` автоматично наповнює in-memory базу тестовими даними (включаючи тестову організацію "SkillForge Demo"). Ви можете увійти, використовуючи наступні облікові дані:

| Роль | Email | Пароль |
|------|-------|--------|
| **Супер Адмін** | `supercurator@skillforge.dev` | `supercurator123` |
| **Куратор Організації** | `curator@skillforge.dev` | `curator123` |
| **Викладач** | `educator@skillforge.dev` | `educator123` |
| **Студент 1** | `learner@skillforge.dev` | `learner123` |
| **Студент 2** | `learner2@skillforge.dev` | `learner123` |

Також `course-service` автоматично наповнює базу готовими курсами (DevOps та Cloud Technologies) разом із уроками та квізами.

### Крок 4. Зупинка
Щоб зупинити всі сервіси, поверніться до вікна, де ви запускали `run_script.bat`, та натисніть будь-яку клавішу. Скрипт (завдяки недавньому оновленню) автоматично закриє всі дочірні вікна та процеси (Java та Node).

## 🔧 Розв'язання відомих проблем

- **CORS / 403 Forbidden**: API Gateway налаштовано для видалення заголовку `Origin` перед пересилкою на мікросервіси. `user-service` та `course-service` обробляють запити як локальні. Не вмикайте CORS всередині мікросервісів — це прерогатива Gateway!
- **Проблема з кирилицею**: Якщо `data.sql` у H2 відображає некоректні символи, переконайтеся, що проєкт зібрано з `UTF-8` (`<project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>` у `pom.xml` та `spring.sql.init.encoding=UTF-8` в `application.yml`).
- **MultipleBagFetchException**: При завантаженні складних сутностей з кількома колекціями (наприклад, `Course` -> `Modules` -> `Lessons`), використовується дефолтний `default_batch_fetch_size: 16` у Hibernate, що запобігає цій помилці та усуває проблему N+1.
